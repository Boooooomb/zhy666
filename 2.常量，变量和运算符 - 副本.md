##### 1.3 原码，反码和补码 

```
计算机他爹 
	冯·诺依曼
	1. 计算机二进制思想
	2. 计算机文件存储关系，万物皆文件

原码，反码和补码 解决了计算机过于复杂的计算过程！！！简化计算机的运算逻辑，提供运算效率。

正整数
	118
	正整数原码，反码和补码是一样的
		都是对应数值的二进制位 0111 0110(2)
负整数
	-118
	原码:
		对应正整数的二进制位，最高位表示符号位 1
		0111 0110(2) ==> 1111 0110(2)
	反码：
		对应原码，按位取反，符号位不变
		1111 0110(2) ==> 1000 1001(2)
	补码：
		反码 + 1
		1000 1001(2) + 1 ==> 1000 1010(2) 
		补码就是负数在内存中最终保存形式
	
	0111 0110(2)   --> 118
	1000 1010(2)   --> -118
     0000 0000   --> 0        
```

##### 3.5 变量定义异常情况总结

###### 3.5.1 给予float类型变量赋值

```
float floatNumber = 3.14;
/*
Demo2.java:16: 错误: 不兼容的类型: 从double转换到float可能会有损失
              float floatNumber = 3.14;
                                  ^
1 个错误
*/
计算机保存数据的原则:
	数据的精度和准确程度高于一切！！！
	3.14是一个浮点类型的常量，对于计算机来说，为了保证数据的完整性和精度问题，会将3.14看作是一个double类型来处理。不考虑是否浪费了内存空间。
	如果是一个浮点类型的常量赋值给float类型的变量，需要在常量之后加上 大写F，声明当前的常量是一个float类型
	修改: float floatNumber = 3.14F;
```
###### 3.5.2 给予long类型变量赋值
```
	同样是为了数据类型一致化，和数据精度的情况考虑，如果赋值给long类型的变量，需要在赋值的常量之后加上大写的L。声明当前数据是一个long类型
	修改: long longNumber = 40L;
```

##### 5.2 自增自减运算符

```
自增自减运算符。已经处于淘汰的边缘，存在过多的歧义性！！！会导致代码阅读性很差！！！
Swift  iOS开发，其中就已经取消了对于自增自减运算符的支持。

自增自减运算符有且只能操作变量，自增操作就是当前变量 + 1，自减操作就是当前变量 - 1

++：
	在变量之后，先执行该行语句，再执行自增操作
	在变量之前，先执行自增操作，再执行改行语句
	
--：
	在变量之后，先执行该行语句，再执行自减操作
	在变量之前，先执行自减操作，再执行改行语句

使用建议:
	1. 使用自增自减运算符，单独成行！！！
	2. 使用 += 1 和 -= 1 替换 自增自减运算符
```
